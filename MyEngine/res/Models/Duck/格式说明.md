## 说明

1. glTF (标准分离式) —— 开发测试首选
这是最基础的格式。

Duck.gltf: 一个 JSON 文件，包含模型的结构、材质定义、节点层级等描述信息。

Duck0.bin: 二进制文件，存储了真实的顶点坐标、法线、索引等几何数据。

DuckCM.png: 贴图文件。

测试重点: 你的导入类是否能根据 JSON 里的相对路径自动找到 .bin 和 .png。

2. glTF-Binary (二进制单文件)
Duck.glb: 这是一个二进制包装格式，把 JSON、BIN 和图片全部打包成了一个文件。

测试重点: 你的类是否支持解析二进制流，并从中拆分出不同的数据块。这是目前游戏引擎（如 Unity/Unreal）最常用的分发格式。

3. glTF-Embedded (Base64 内嵌式)
Duck.gltf: 打开这个文件你会发现它非常大。因为它把图片和二进制数据全都转成了 Base64 字符串 编码直接写在了 JSON 里面。

测试重点: 你的解析器是否具备处理 Data URI（Base64 编码数据）的能力，无需读取外部文件。

4. glTF-Draco (压缩式)
Duck.bin: 这里的 .bin 经过了 Google 的 Draco 几何压缩算法 处理，体积非常小。

注意: 这种格式需要你的导入类集成 Draco 解压库，否则无法直接读取顶点数据。

5. glTF-Quantized (量化式)
特点: 将顶点的浮点数（float）量化为整数（如 short），以减小文件体积并提高传输效率。

测试重点: 验证你的解析器是否能正确处理材质扩展中的 KHR_mesh_quantization 属性。
